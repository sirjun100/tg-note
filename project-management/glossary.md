# Project Management Glossary

Terminology and definitions for consistent project-management language.

## Backlog Items

| Term | Definition |
|------|------------|
| **User Story** | A feature or capability from the user's perspective. ID format: US-XXX. |
| **Defect** | A bug or incorrect behavior that needs fixing. ID format: DEF-XXX. |
| **Technical Debt** | Refactoring, cleanup, or infrastructure work. ID format: TD-XXX. |
| **Retrospective Improvement** | Process improvement from sprint retrospectives. ID format: RI-XXX. |

## Status

| Symbol | Status | Meaning |
|--------|--------|---------|
| ⭕ | Not Started | Item not yet started |
| ⏳ | In Progress | Item currently being worked on |
| ✅ | Completed | Item finished and verified |

## Priority

| Symbol | Level | Meaning |
|--------|-------|---------|
| 🔴 | Critical | Must be fixed/implemented immediately |
| 🟠 | High | Should be addressed soon |
| 🟡 | Medium | Nice to have, can wait |
| 🟢 | Low | Future consideration |

## Process Terms

| Term | Definition |
|------|------------|
| **Sprint** | Time-boxed iteration (e.g. 2 weeks) for delivering work. |
| **Story Points** | Relative estimate (Fibonacci: 1, 2, 3, 5, 8, 13). |
| **Definition of Done** | Quality gate criteria for completed work. |
| **Definition of Ready** | Criteria for backlog items before sprint assignment. |
